# Plan to Fix PDF Generation Issues

## Overview
The PDF generation in `app/services/pdf_service.py` has two issues:
1. Dynamic locations are not populated in the PDF - only hardcoded locations are shown
2. Some phone numbers are missing the leading '0' prefix in the display

## Issue Analysis

### Dynamic Locations Issue
- Current code uses hardcoded valid locations: `{'kanana', 'majaneng', 'mashemong', 'soshanguve', 'kekana'}`
- New dynamic locations are added as tags but not recognized by the PDF generator
- Locations are stored as tags in contact metadata, with dynamic locations being any tag that's not a role tag or 'member'

### Phone Number Formatting Issue
- The `format_phone_for_display` function handles some cases but may not cover all scenarios where the leading '0' is missing
- Current logic only adds '0' for 9-digit numbers starting with 7,8,9
- Some phone numbers might be stored as 10-digit numbers without the '0' prefix

## Proposed Solution

### 1. Fix Dynamic Locations
**File: `app/services/pdf_service.py`**
- Modify `extract_location_from_tags()` to recognize dynamic locations
- Exclude known role tags: 'pastor', 'protocol', 'worshiper', 'usher', 'financier', 'servant', 'member'
- Return the first tag that is not in the excluded list (assuming it's a location)

**Code Changes:**
```python
def extract_location_from_tags(tags: List[str]) -> str:
    # Exclude role and membership tags - everything else is a location
    excluded_tags = {'member', 'pastor', 'protocol', 'worshiper', 'usher', 'financier', 'servant'}
    
    if not tags:
        return ''
    
    for tag in tags:
        tag_lower = tag.lower()
        if tag_lower not in excluded_tags:
            return tag.capitalize()
    
    return ''
```

### 2. Fix Phone Number Formatting
**File: `app/services/pdf_service.py`**
- Enhance `format_phone_for_display()` to handle more phone number formats
- Add logic to ensure mobile numbers have the '0' prefix
- Normalize phone numbers before formatting

**Code Changes:**
```python
def format_phone_for_display(phone: str) -> str:
    if not phone:
        return ''
    
    phone = phone.strip()
    
    # Remove international prefix if present
    if phone.startswith('+27'):
        phone = phone[3:]
    elif phone.startswith('27') and len(phone) > 10:
        phone = phone[2:]
    
    # Ensure mobile numbers have leading 0
    if len(phone) == 9 and phone[0] in ['7', '8', '9']:
        phone = '0' + phone
    elif len(phone) == 10 and phone[0] in ['7', '8', '9']:
        # Handle case where 10-digit number is missing 0 (shouldn't happen but defensive)
        phone = '0' + phone
    
    # Format with spaces
    if phone.startswith('0') and len(phone) == 10:
        return f"{phone[:3]} {phone[3:6]} {phone[6:]}"
    
    # Return as-is if format doesn't match expected patterns
    return phone
```
