# Plan: Prevent Attendance Duplicates

## Problem Analysis
- Attendance records are appearing with exact duplicates (same name, phone, location, member status) in PDF reports
- Existing duplicates in database need to be cleaned before enforcing uniqueness
- Current constraint on full service_date allows multiples if times differ, but business rule is one attendance per contact per service per day
- Race conditions in service logic could allow duplicates
- Issue stems from Flutter app sending duplicate requests

## Solution Overview
Clean existing duplicate attendances, then implement database-level uniqueness constraint based on date (not datetime) to prevent future duplicates. Update service logic accordingly.

## Implementation Steps

### 1. Clean Existing Duplicates
- Identify and remove duplicate attendances where same contact_id, service_type, and date(service_date)
- Keep the earliest recorded attendance for each group
- Run cleanup script or migration to remove duplicates

### 2. Update Attendance Model
- Remove previous UniqueConstraint
- Add unique constraint or index on (contact_id, service_type, date(service_date)) to prevent duplicates per day

### 3. Create Database Migration
- Generate Alembic migration to clean duplicates and add the date-based unique constraint

### 4. Update Attendance Service
- Ensure the check uses date comparison consistently
- Handle IntegrityError for the new constraint

### 5. Testing
- Verify no duplicates in existing data
- Test duplicate prevention with API calls
- Confirm PDF reports show unique records

### 6. Client-Side Note
- Inform development team to fix Flutter app to prevent sending duplicate attendance requests

## Files to Modify
- app/models.py: Update UniqueConstraint to use date-based uniqueness
- app/services/attendance_service.py: Ensure consistent date checking
- Alembic migration file: New migration to clean duplicates and add date-based constraint

## Potential Risks
- Data loss from removing duplicate attendances (mitigate by backup)
- Constraint prevents legitimate multiple attendances if times differ but same day (confirmed business rule allows only one per day)

## Validation
- Existing duplicates removed from database
- PDF reports show unique records
- Attempt to record duplicate fails with error
