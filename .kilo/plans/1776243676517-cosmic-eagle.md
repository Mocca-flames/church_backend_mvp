# Fix Phone Number Formatting in PDF Generation

## Problem
The phone number "647764004" is displayed as "647764004" in the PDF, but it should be formatted as "064 776 4004" (with leading 0 and spaces).

## Root Cause
In `app/services/pdf_service.py`, the `format_phone_for_display` function only adds a leading "0" to 9-digit phone numbers starting with "7", "8", or "9". South African mobile numbers can also start with "6" (Telkom Mobile), so the condition needs to include "6".

## Solution
Update the condition in `format_phone_for_display` to include "6" in the list of digits that trigger adding a leading "0" for 9-digit numbers.

## Steps
1. Edit `app/services/pdf_service.py`
2. Change line 93 from: `if len(phone) == 9 and phone[0] in ["7", "8", "9"]:`
3. To: `if len(phone) == 9 and phone[0] in ["6", "7", "8", "9"]:`

- Confirm it displays as "064 776 4004" in the PDF
